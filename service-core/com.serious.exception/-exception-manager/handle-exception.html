handleException

fun handleException(exception: Throwable): Throwable

handles the specified exception with respect to registered handlers. The following logic is implemented:

  • execute the chain of matching log methods starting with the most specific. Every log-method is free to call ExceptionManager.proceed that will call the next best matching method.

  • execute the chain of matching xform methods starting with the most specific. Every xform-method is free to call ExceptionManager.proceed that will call the next best matching method. Any explicit throw statement will terminate the chain and the toplevel handleException call. Otherwise the resulting - possibly transformed - objects will be returned.

If no matching xform method can be identified the call will throw a MethodDispatcher.NoApplicableMethodError.

Return

the - possibly transformed - exception

Parameters

exception

the original exception.
